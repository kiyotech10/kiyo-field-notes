# 🛸 Antigravity（Gemini）ブログ運営・執筆マニュアル

ようこそ！このファイルは、AIアシスタントの **Antigravity（Gemini）** と連携して、このブログ「Kiyo's Field Notes」を最も簡単かつクリエイティブに運営していくための仕様書 兼 ガイドラインです。

今後ブログの更新、新機能の追加、デザインの調整を行いたいときは、このガイドを参考にチャットで指示を出すだけで完結します。

---

## 1. 執筆＆デプロイのカンタン自動ワークフロー

読者に向けて記事を書き、公開するまでの手順は非常にシンプルです。

### 📥 ステップ 1: アイデアや下書きの準備
デスクトップに下書き用のテキスト（例：`ブログ.txt`）を用意するか、チャットに直接「今日の出来事のメモ」を貼り付けます。
* 例：箇条書きのメモ、外食の感想、英語の勉強記録、TOEICの感想など、まとまっていなくてもOKです。

### ✍️ ステップ 2: Antigravityに執筆とデプロイを指示する
チャットで以下のように伝えるだけで、記事の執筆から本番公開までを自動で行います。

**指示プロンプトの例：**
> 「デスクトップの `ブログ.txt` をもとに、UNIQLOのおすすめ商品の紹介記事を **Lifehack** カテゴリで書いて。できたら、GitHubに反映（デプロイ）まで完了させておいて！」

### ⚙️ ステップ 3: 自動化される処理（裏側）
1. **記事ファイルの自動生成**：`src/content/{カテゴリ}/` に、適切なFrontmatter（タイトル・日付・スラグなど）付きの `.mdx` または `.md` ファイルを生成します。
2. **ビルドテスト**：`npm run build` をローカル環境で自動実行し、プログラムエラーがないか事前検証します。
3. **GitHubへのプッシュ**：`git add`, `git commit`, `git push` を自動実行します。
4. **Vercel自動デプロイ**：Vercelが自動ビルドを実行し、**数分以内に本番サイトに公開完了**します。

## 2. ブログのカテゴリ一覧とスラグ

記事を書く際は、以下の6つのカテゴリから最も適したフォルダに配置されます。

| フォルダ名 | カテゴリ名 (ラベル) | 推奨するコンテンツ内容 | Googleパステル配色 |
| :--- | :--- | :--- | :--- |
| `src/content/nursing/` | **Nursing** (看護) | 看護の仕事、病棟勤務、キャリア、看護制度の考察 | ティール (Teal) |
| `src/content/english/` | **English** (英語) | TOEIC対策、単語・文法の学習ノート、シャドーイング記録 | ブルー (Blue) |
| `src/content/lifehack/` | **Lifehack** (効率化) | 効率的お金管理、おすすめガジェット、UNIQLO、時短術 | グリーン (Green) |
| `src/content/study/` | **Study** (勉強) | 社労士試験対策、その他資格試験の勉強記録 | パープル (Purple) |
| `src/content/reading/` | **Reading** (読書) | 読んだ本や論文の要約、考察、書評ノート | オレンジ (Orange) |
| `src/content/diary/` | **Diary** (日常) | 旅行（沖縄など）、日々の他愛もない日記や思考整理 | コーラル (Coral) |

## 3. 記事のFrontmatter（ヘッダー情報）仕様

記事ファイルを作成する際、ファイルの冒頭に自動で以下のメタデータを挿入します。

```yaml
---
title: "記事のタイトル"
description: "検索エンジンやSNSで表示される、記事の魅力的な要約"
pubDate: 2026-05-24         # 投稿日（自動で本日の日付が入ります）
updatedDate: 2026-05-25     # 更新日（任意：リライト時に自動追記します）
heroImage: ./image.jpg      # アイキャッチ画像（任意：記事フォルダ内に置いた相対パス）
featured: false             # Trueにするとトップページの最上部「Featured」に固定表示されます
---
```

## 4. よく使う運用・保守用の開発コマンド

もしご自身でテストやプレビューを行いたい場合は、プロジェクトフォルダ内で以下のコマンドを実行できます。
（これらもチャットで「開発サーバーを立ち上げて」などと私に指示すれば、代わりに起動します）

```bash
# 依存関係のインストール（環境構築時）
npm install

# ローカル開発サーバー起動（http://localhost:4321 で確認可能）
npm run dev

# 本番仕様の静的ビルドテスト（エラーの事前検知に最適）
npm run build

# ビルドされた本番データ（./dist）のローカルプレビュー
npm run preview
```

## 5. 今後追加できるおすすめ機能アイデア 💡

もしブログをさらに拡張したくなったら、いつでもお声がけください。以下の機能も即座に実装可能です！

* **Fuse.js によるクライアントサイド検索機能**（カテゴリをまたいだ超高速キーワード検索）
* **タグ機能の追加**（Frontmatterに `tags: ["TOEIC", "UNIQLO"]` を追加し、タグ別一覧ページを作成）
* **関連記事表示エリア**（各記事の末尾に、関連する別の記事を3つ自動でサジェスト）
* **Satori を使った動的OGP画像の自動生成**（記事タイトルから、SNSシェア用の画像を自動生成）
